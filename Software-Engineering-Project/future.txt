🧩 PROJECT CONTEXT PROMPT (COPY THIS)

Project Name: Virtual Coupon Distribution System (VCDS)

🧾 GOAL

We are building a full-stack web application for Virtual Coupon Distribution and Redemption (VCDS) using:

Frontend: React (via create-react-app)

Backend: Node.js + Express

Database: MySQL

APIs: RESTful routes for campaigns, coupons, redemptions, and reports

The goal is to implement the system step by step according to five Jira Epics and their respective user stories.

📘 JIRA EPICS & USER STORIES
🟩 EPIC 1: Coupon Campaign Management

Epic Description: Manage creation, updates, and scheduling of coupon campaigns.

User Stories:

Create Coupon Campaign

Edit or Update Active Campaigns

View Campaign List & Status

Schedule Campaign Activation

Validate Campaign Inputs

🟩 EPIC 2: Coupon Code Generation & Security

Epic Description: Ensure coupon codes are unique, random, and secure.

User Stories:

Generate Unique Coupon Codes

Ensure Code Randomness

Encrypt Coupon Data

Apply Code Expiration Rules

Prevent Duplicate Code Generation

🟩 EPIC 3: Coupon Distribution Module

Epic Description: Distribute coupons through multiple channels (Email, SMS, QR, etc.)

User Stories:

Distribute Coupons via Email

Distribute Coupons via SMS

Generate and Share QR Codes

Retry Failed Deliveries

Integrate APIs for Communication

🟩 EPIC 4: Coupon Redemption & Fraud Detection

Epic Description: Implement secure and validated redemption flow.

User Stories:

Validate Coupon on Redemption

Prevent Duplicate Redemptions

Log Redemption Attempts

Detect Fraudulent Patterns

Handle Expired or Invalid Coupons

🟩 EPIC 5: Reporting, Analytics & Access Control

Epic Description: Provide analytics, reports, and user-role-based access control.

User Stories:

Generate Merchant Reports

View Redemption Analytics Dashboard

Apply RBAC Permissions

System Performance Validation

Export Reports in Multiple Formats

Main APIs Implemented So Far
Endpoint	Method	Description
/api/coupons/generate	POST	Generate new coupons (with unique codes, insert into DB)
/api/coupons/redeem	POST	Redeem coupon, mark as used, and log redemption
/api/campaigns	GET	Get all campaigns
/api/campaigns	POST	Create new campaign
/api/reports/merchant/:id	GET	Generate merchant report showing issued vs. redeemed coupons
✅ MySQL Schema (Current)

Database: vcds

Tables:

users — Stores users (admin, merchant, customer)

campaigns — Coupon campaign info

coupons — Coupon codes with status (redeemed / not redeemed)

redemptions — Redemption logs

delivery_attempts — Tracks email/SMS send attempts

merchant_stats — Aggregate data for reports

notifications — Messages sent to users

system_logs — Admin and merchant actions

All tables have been verified and foreign keys are correctly linked.

🔑 Sample Backend Flow Tested

✅ Generate Coupons

curl -X POST http://localhost:5000/api/coupons/generate \
  -H "Content-Type: application/json" \
  -d "{\"campaignId\":1,\"count\":3,\"channel\":\"email\",\"recipients\":[\"customer1@example.com\",\"customer2@example.com\",\"customer3@example.com\"]}"


Response:

{"createdCount":3,"coupons":[{"id":11,"code":"8HPHIVVP"}, ... ]}


✅ Redeem Coupon

curl -X POST http://localhost:5000/api/coupons/redeem \
  -H "Content-Type: application/json" \
  -d "{\"code\":\"8HPHIVVP\",\"userIdentifier\":\"customer1@example.com\"}"


Response:

{"message":"Coupon redeemed","couponId":11,"campaignId":1}


✅ MySQL Verification

coupons.redeemed → updated to 1

new entry in redemptions for audit

delivery_attempts entries created on generation
